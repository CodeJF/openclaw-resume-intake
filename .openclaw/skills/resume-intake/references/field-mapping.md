# 简历录入字段映射

## 已批准的安全字段

固定生产链路里，只填写这些可靠度足够高的字段：

- 应聘者姓名
- 年龄
- 应聘岗位
- 联系方式
- 学历
- 毕业院校
- 专业
- 是否为全日制
- 最近一家公司名称
- 目前薪资
- 期望薪资
- 附件

除非业务规则被明确扩展，否则其他字段一律留空。

## 提取原则

- 不得编造。
- 宁可留空，也不要填错。
- 保持保守归一化。
- 如果薪资文本有歧义、不是数字、或包含 `面议` / `保密` / `详谈` 等字样，薪资字段留空。

## 当前启发式规则

### 姓名

优先匹配明确的 `姓名:` 模式；兜底时可使用简历前几行里独立出现的中文姓名行。

### 联系方式

若能提取到手机号和邮箱，则组合成 `phone / email`；只命中一个时，只返回那一个。

### 年龄

只使用明确的 `年龄:` 模式。

### 学历

当前关键词扫描顺序：

- 博士
- 硕士
- 本科
- 大专
- 中专
- 高中

### 毕业院校

使用以 `大学` 或 `学院` 结尾的保守匹配模式。

### 专业

优先匹配明确的 `专业:` 模式。

### 是否全日制

映射规则：

- `全日制` => `是`
- `非全日制|成人教育|自考|函授` => `否`

### 最近一家公司名称

优先匹配 `最近一家公司` / `最近公司` / `现公司` / `就职于` 等显式模式。

### 应聘岗位

优先匹配 `应聘岗位` / `求职意向` / `意向岗位` 等显式模式。

必要时，需要从同一行中剔除后续无关标签，例如：

- 意向城市
- 期望薪资
- 电话
- 邮箱
- 性别
- 年龄
- 现所在地
- 最高学历

## 输出格式

只返回有值的字段，不输出空字符串字段。

示例：

```json
{
  "应聘者姓名": "王楠",
  "联系方式": "13800000000 / test@example.com",
  "学历": "本科"
}
```
