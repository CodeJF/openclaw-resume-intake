# 强护栏规则

## 附件要求补充
在当前简历录入既定目标中，原始简历 PDF 应作为业务必需件写入 `附件` 字段。

### 这意味着
- 不能把“只写了文字字段”默认当成整条录入成功。
- 如果 `附件` 未写入成功，必须向用户明确反馈。
- 在后续实现中，附件上传步骤应被视为标准流程的一部分，而不是可选尾步骤。


## create 完成不等于整条流程完成
在当前简历录入目标里，record.create 只是第一步。
如果附件尚未补传完成，不得对用户宣称“已录入完成”。


## 正式链路约束
在当前简历录入目标里，create 成功后必须继续推进附件补传，不允许在中间提前结束。
