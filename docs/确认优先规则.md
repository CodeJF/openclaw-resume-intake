# 确认优先规则

适用范围：本 workspace 中所有与飞书用户交互、简历录入、多维表格写入、目标注册相关的流程。

## 核心原则
凡是不确定的事情，一律不要直接操作；必须先向飞书用户确认，拿到明确结果后再执行。

## 必须先确认的场景
包括但不限于：
- 不清楚要写入哪个多维表格
- 不清楚是哪个数据表 / 视图 / 业务目标
- 不清楚是否应该新增 target_key
- 不清楚字段应如何映射
- 不清楚是新增、更新、还是忽略
- 不清楚是否允许写入附件
- 不清楚是否要覆盖已有记录
- 不清楚用户真正想做的是哪一步

## 禁止行为
在信息不完整、目标不明确、用户意图不清楚时，禁止：
- 猜测用户要写入的多维表格
- 根据业务名称直接推断 app_token / table_id
- 因为名字相似而自行选择目标表
- 自动创建新表来“兜底”
- 默认执行更新、覆盖、补写等操作

## 正确做法
1. 先明确告诉用户当前缺什么信息。
2. 直接反问用户，获取明确答案。
3. 只在拿到明确答案后再继续执行。
4. 若仍无法确认，则停止操作，而不是猜测执行。

## 目标注册规则
如果一个新的多维表格目标尚未注册：
1. 先让用户明确说明要写入哪个多维表格。
2. 最好让用户提供链接、名称、表名、视图名等可核对信息。
3. 核验真实 app_token / table_id 后，再写入 target 配置。
4. 未确认前，不得直接写入，也不得创建。

## 记忆记录规则
- 所有这类流程约束、偏好、记忆内容，统一使用中文记录。

## 一句话版本
不确定，就先问；确认后，再执行。
