# 轻量去重方案（建议）

## 目标
在正式创建记录之前，尽量识别“同一候选人重复录入”，减少同一份简历或同一联系方式被反复创建。

## 推荐最小策略
优先使用以下键做轻量匹配：
1. 联系方式（手机号优先，其次邮箱）
2. 应聘者姓名 + 联系方式
3. 若无联系方式，则退化为 应聘者姓名 + 最近一家公司名称

## 推荐执行顺序
1. 提取字段
2. 如果没有 `应聘者姓名`，直接不做去重，进入人工判断/继续录入
3. 如果有 `联系方式`，先按 `联系方式 is <value>` 查询
4. 若命中，再比较 `应聘者姓名`
5. 若高度匹配：
   - 默认提示“疑似已存在”
   - 允许用户决定：新建 / 覆盖字段 / 仅补附件
6. 若无命中，再正常 create

## 当前实现建议
- 当前阶段先不自动阻断 create
- 先增加“录入前提示可疑重复”的能力
- 待线上观察稳定后，再考虑默认阻断重复 create

## 注意事项
- 表中已有“🛎 重复人选提醒”公式，但这是事后提醒，不是录入前幂等控制
- 若后续要自动查重，建议使用 `feishu_bitable_app_table_record.list` + 结构化 filter
- 若要更强幂等，可追加文件 hash / message_id / sender_id 作为辅助键
